## 任务日志

**任务日志**用于查看异步类能力的执行记录，例如 Midjourney 绘图、Suno 音乐生成、视频生成等：每条记录对应一次提交，可跟踪排队、进行与结束状态，并在成功后查看产出或失败原因。

进入方式：侧栏 **任务日志**，或访问 `/console/task`。

![任务记录：筛选与列表](/settings/task/task-records.png)

### [列表与筛选](#列表与筛选)

页面上方通常会展示与当前时间范围相关的 **消耗额度、RPM、TPM** 等摘要（以实际控制台为准）。下方为任务表格，常见列包括：

| 列 | 含义 |
| --- | --- |
| 提交时间 / 结束时间 | 任务入队与完成时刻。 |
| 花费时间 | 从提交到结束所耗时长。 |
| 平台 / 类型 | 任务所属服务与业务类型。 |
| 任务 ID | 唯一标识，便于在支持渠道侧对账或排障。 |
| 任务状态 / 进度 | 当前生命周期阶段与百分比等。 |
| 详情 | 展开可查看更完整的请求与返回信息（若有）。 |

在筛选区选择 **时间范围**、填写 **任务 ID** 后点击 **「查询」**；**「重置」** 可清空条件。**「列设置」** 可按个人习惯隐藏或固定部分列；**「紧凑列表」** 可切换更紧凑的行高以便一屏浏览更多行。

### [任务状态说明](#任务状态说明)

不同产品线的状态文案可能略有差异，常见含义如下：

| 状态 | 说明 |
| --- | --- |
| `PENDING` | 已提交，等待调度或排队中。 |
| `IN_PROGRESS` | 正在生成或处理中。 |
| `SUCCESS` | 已完成，可在详情或关联入口查看结果。 |
| `FAILURE` | 失败结束；若平台支持自动退费，配额会按规则退回（以实际提示为准）。 |

若长时间停留在 `PENDING` 或 `IN_PROGRESS`，可先确认上游渠道与账户额度是否正常，再结合 **任务 ID** 联系支持或管理员排查。
